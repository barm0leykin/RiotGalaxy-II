# Product Requirements Document (PRD) — RiotGalaxy II

## 1. Обзор

**RiotGalaxy II** — 2D космический шутер в стиле Galaga / Space Invaders / Radiant: игрок управляет
кораблём в нижней части экрана, отражая волны врагов, которые строятся в формации-ульи и
пикируют в атаку. Игра кросс-платформенная (десктоп и Android), написана на **C# + MonoGame**.

Проект развивается из завершённой миграции с CocosSharp на MonoGame (история — `MIGRATION.md`).
Этот PRD описывает игру **как самостоятельный продукт** и задаёт рамки дальнейшего развития —
без привязки к старому движку.

## 2. Видение и цели

- **Видение:** аркадный шутер с «живым» роем врагов (Galaga-механика вылетов), понятным
  управлением и плавным ростом сложности; легко расширяемый контентом через конфиги.
- **Цели:**
  - Затягивающий core-loop: отражай волны → собирай очки/бонусы → прокачивай оружие → босс.
  - Контент (враги, уровни, оружие, бонусы) задаётся данными (YAML), а не кодом.
  - Один код — две платформы (десктоп для разработки, Android как основная цель).
- **Не-цели (сейчас):** мультиплеер, онлайн-сервисы, 3D, внутриигровые покупки, iOS.

## 3. Платформы и технологии

- **Язык/движок:** C#, MonoGame (Microsoft.Xna.Framework), OpenGL (DesktopGL) / Android.
- **Целевые платформы:** Windows, Linux, macOS (DesktopGL), **Android** (net9.0-android).
  iOS — потенциально в будущем, сейчас не поддерживается.
- **Разрешение:** виртуальный кадр 1280×768; на экранах иной формы — letterbox
  (масштаб с сохранением пропорций).
- **Среда разработки:** VSCode/Windsurf под Linux; отладка desktop по F5.

## 4. Игровой процесс

**Управление:**
- Десктоп: клавиатура (←/→ или A/D — движение, Space — огонь, 1/2/3 — оружие, Esc/P — пауза).
- Android: тач (палец двигает корабль), **автоогонь**, системная кнопка «Назад».

**Бой и оружие:**
- 3 вида оружия с уровнями прокачки: пушка, пулемёт (разброс), лазер (пробивающий).
- Снаряды игрока и врагов; столкновения (таран врага, попадания).

**Враги и ИИ:**
- Типы: scout / blue / green / red (+ босс), параметры из `enemies.yaml`.
- **Машина состояний** (влёт → роение → атака) для «свободных» врагов.
- **Улей (Galaga):** враги выстраиваются в формацию вверху и синхронно барражируют; из улья
  по очереди срываются в пике-атаку и возвращаются в свою ячейку. Тактики пике:
  `random` / `snake` / `ram` (таран) / `ellipse` (петля); скорость атаки — отдельный параметр.
  В строю враги не стреляют (огонь только в вылете).

**Бонусы и награды:** HP, апгрейд оружия, «нюк» (убить всех), звёзды-очки с магнитом
(оборудование корабля, настраивается в `weapons.yaml`).

**Уровни/прогрессия:** кампания с нарастанием (формации, маршруты-«змейки», синхронные
параллельные волны, босс в финале). Игрок и счёт переносятся между уровнями;
победа/поражение → итог и рестарт.

**UI:** заставка, главное меню, настройки (громкость), экран между уровнями, HUD
(HP/очки/оружие), всплывающие сообщения.

## 5. Функциональные требования

- Полный игровой цикл: меню → заставка → уровни → победа/поражение → рестарт.
- Спавн врагов по таймлайну уровня; формации, маршруты, параллельные волны, вылеты из улья.
- Конфигурируемость данными: оружие, враги, бонусы, опции игрока, уровни, маршруты — в YAML,
  меняются без перекомпиляции; у каждого конфига есть дефолты-фолбэк.
- Сохранение пользовательских настроек (громкость) на диск.
- Кросс-платформенная загрузка ассетов (десктоп и Android из одного кода).

## 6. Нефункциональные требования

- **Производительность:** стабильные ~60 FPS на целевых устройствах; без задержек звука и
  рывков анимации.
- **Сборка Android — только в Docker:** хост-система не засоряется (SDK/JDK/workload — в образе).
- **Кросс-платформенность:** единая логика в `RiotGalaxy.Core`; платформенные обёртки тонкие.
- **Конфигурируемость:** игровой баланс и контент — в данных (YAML), не в коде.
- **Поддерживаемость:** архитектура документирована (`ARCHITECTURE.md`); по мере изменений
  обновляются `ARCHITECTURE.md` и `TODO.md`.
- **Логирование/отладка:** `Log.Debug/Error` → консоль и `riot.log` (Android — logcat, тег DOTNET).

## 7. Архитектура (кратко)

Паттерны: **Strategy** (компоненты движения/стрельбы/столкновений, оружие),
**Command** (действия/кнопки), **State** (состояния игры и ИИ врагов). Центральный
`GameManager` (цикл/состояния/столкновения/отрисовка), `World`/`Hive`/`Route` для построений.
Подробно — в `ARCHITECTURE.md`.

## 8. Направление развития (roadmap)

Краткосрочно (полировка «сочности» и контента):
1. Визуальные взрывы/частицы, отдельный HUD-класс.
2. Фоновая музыка и звук подбора бонусов.
3. Балансировка кампании; раздать вылеты из улья по уровням с ростом темпа.

Среднесрочно:
4. Новые враги/боссы и паттерны, новые тактики пике и маршруты.
5. Сохранение прогресса/рекорды; кат-сцены между волнами.
6. CI-сборка APK; release-подпись.

## 9. Критерии успеха

- Завершаемая кампания на десктопе и Android без критических багов.
- Контент расширяется правкой YAML без изменения кода.
- Стабильный FPS и отзывчивое управление на обеих платформах.
- Сборка Android воспроизводима в Docker; десктоп — одной командой.
